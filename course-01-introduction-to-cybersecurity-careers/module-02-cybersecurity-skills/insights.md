# Module 2 : Cybersecurity Skills

## IT Fundamentals and Technical Skills

- Strong IT fundamentals = foundation for cybersecurity
- Key areas:
  - hardware
  - operating systems
  - networking
  - programming
  - databases
  - cloud
  - automation

## Hardware

- Helps identify hardware-related threats/vulnerabilities
- Important for preventing unauthorised access and protecting data
- Useful to understand:
  - hardware components
  - configuration
  - common failures
  - hardware security measures

## Operating Systems

- Cyber professionals need to understand:
  - user privileges
  - updates and patches
  - file systems
  - processes and threads
  - memory management
  - OS logging and monitoring
  - system behaviour
- Helps detect abnormalities, manage vulnerabilities, and improve defences

## Networking

- Understand how devices communicate and data moves between systems
- Cyber threats often target network weaknesses
- Important foundations:
  - network types and structures
  - protocols
  - OSI model
  - troubleshooting
  - network monitoring tools
  - firewalls and IDS
  - wireless securit
- Needed to:
  - recognise suspicious network behaviour
  - identify potential attacks
  - apply preventive measures

## Programming

- Cyber professionals do not always need to be devs
- Coding basics help:
  - automate tasks
  - understand attack vectors
  - identify software vulnerabilities
  - create scripts/tools for testing
  - read and interpret code
  - understand secure coding
  - debug software
  - understand web programming basics

## Databases

- Databases often store sensitive data
- Need to understand how data is stored, retrieved, and protected
- Useful areas:
  - SQL
  - NoSQL
  - database schemas
  - encryption
  - permissions and roles
  - backups
  - restoration
  - replication

## Cloud

- Need understanding of:
  - cloud services
  - IaaS, PaaS, SaaS
  - public, private, hybrid, and community clouds
  - shared responsibility model
  - cloud security best practices

## Automation

- Automates routine security and network tasks
- Helps improve efficiency and response
- Cyber professionals need to understand both benefits and security risks

## Key Takeaway

- IT fundamentals help cyber professionals identify, prevent, and mitigate threats
- Strong cyber ability depends on understanding the systems being protected

- Showing cyber interest outside course content can help stand out
- Useful activities:

  - Capture the Flag (CTF) events
  - independent projects

## Technical Foundations

- Strong programming foundation useful
- Important skills:
  - readable code
  - debugging
  - testing

- Understand industry tools for:
  - collaboration
  - version control
  - CI/CD

## Understanding Systems

- Need to understand how systems work before understanding how they might fail
- Broad IT knowledge helps, Coding not always required, but more experience helps understanding
  - how vulnerabilities are introduced and mitigated

## New Technology
- Stay aware of emerging technologies as they create new attack surfaces

## Importance of Cybersecurity Technical Skills

- Key technical areas:
  - information security and assurance
  - security operations
  - cryptography
  - risk management
  - threat management
  - vulnerability management
  - authentication
  - network security

## Information Security and Assurance

- Protects digital and non-digital information
- Focuses on CIA

- Can involve:
  - security policies
  - access controls
  - audits
  - compliance

## Security Operations

- Identifies, investigates, and resolves security incidents
- Includes:
  - monitoring security infrastructure
  - incident response
  - working with other teams

## Cryptography

- Protects information at rest and in transit using secure transformations
- Helps provide CIA

- Important concepts:
  - symmetric/asymmetric encryption
  - hashing
  - digital signatures
  - key management
  - algotithms 


## Risk Management

- Identifies, assesses, and prioritises risks
- Develops strategies to reduce impact
- Monitors whether controls are effective

## Threat Management and Threat Hunting

-  Helps prevent, detect, and respond to threats
- Threat hunting proactively searches for threats missed by automated tools
- Uses:
  - normal vs abnormal behaviour
  - threat intelligence
  - current threat data

## Vulnerability Management

- Continuously finds, prioritises, and resolves vulnerabilities
- Includes:
  - vulnerability scanning
  - penetration testing

## Authentication
- Verifies identity of user, device, or system to ensure only authorised access

## Digital forensics 
- includes :
 - Chain of custody 
 - Data acquisition 
 - Data recovery and preservation 

## Expert Viewpoints: Cybersecurity Technical Skills

- Contributing to open-source projects can help build skills and stand out
- Exposure to enterprise security tools is valuable
- CTFs useful for:
  - learning attacker methods
  - discovering security tools
  - understanding how to triage vulnerabilities
  - building networks with others

## Core Skills

- Understand:
  - common security issues
  - possible causes
  - common exploitation methods
  - why different security tools are useful
- Reading code to spot vulnerabilities

## Critical Thinking

- Ask:
  - how does this system work?
  - how might it fail?
  - what would I do if I were the attacker?
  - where does the system rely on implicit trust?

## Role-Specific Skills
- Skills depend on target area
- Examples mentioned:
  - hacking/testing = integration testing
  - implementation = understanding firewalls
  - engineering = application security
